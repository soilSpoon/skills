# UX 라이팅 한국어 기계 규칙 — 해요체부터 용어집까지

출처: 토스 UX 라이팅 가이드(developers-apps-in-toss.toss.im/design/ux-writing.html).
프린시플이 *방향*이라면 이건 ko 문구를 다듬는 **구체적 치환 규칙**이다. 단, 끝의 **예외
표**를 먼저 익혀라 — 기계적 치환은 토스 가이드 자신이 금지한다.

## 1. 해요체로 통일
제품 안의 **모든** 문구는 해요체로 쓴다. "~합니다"(하십시오체)도, "~해"(반말)도 아니다.
일관된 말투가 일관된 사용자 경험을 만든다.
- "선택하십시오" / "선택해" → "선택해요" · "선택하세요"
- 검증/안내 메시지도 예외 없이: "부정확할 수 있습니다" → "부정확할 수 있어요"

## 2. 능동형 우선
주어를 사용자(또는 행위자)로 두고 능동으로 쓴다. 수동형은 책임 주체를 흐리고 번역체가 된다.
- "송금이 완료됐어요" → "송금했어요"
- "~었/되어"가 보이면 능동으로 바꿀 수 있는지 본다: "되어요" 표기는 "**돼요**"로.

## 3. 긍정형 우선
"안 돼요 / 없어요 / 못 해요"를 가능한 한 **할 수 있는 것**으로 뒤집는다. 특히 에러·제약 안내.
- "쿠폰이 없어요" → "쿠폰을 받으면 쓸 수 있어요"
- "~해야 해요"(강요) → "~할 수 있어요"(권유) — 프린시플 6과 직결.

## 4. 높임말 절제 (과잉 존대 제거)
정중함과 과잉 존대는 다르다. 사물·시스템을 높이거나 사용자에게 굽실대지 않는다.
- "확인하시겠어요?" → 단정적 행동이면 "확인해요" / "확인하기"
- "계시다" → "있다" · "여쭈다" → "확인하다 · 묻다" · "~께" → "~에게"
- 사물 존대 금지: "금액이 부족하십니다" → "금액이 부족해요"

## 5. 명사 나열(한자어 스택) 회피
한자어 명사를 쌓지 말고 동사로 풀거나 "{명사}가 {명사}해서" 형태로 끊는다.
- "결제 실패 사유 확인 필요" → "결제가 실패한 이유를 확인해 주세요"
- "지원금 신청 자격 미달" → "아직 지원금을 신청할 수 없어요"

## 6. 버튼 라벨 규칙
- 좌측(이탈) 버튼은 **"닫기"**, **"취소" 금지**. "취소"는 *진행 중인 작업의 취소*로 오해된다.
  (예: 송금 다이얼로그의 "취소"는 "송금을 취소"로 읽힌다 → "닫기"로.)
- 버튼은 **동사**로, 결과를 예고: "확인" 같은 막연함보다 "보내기 · 결제하기 · 결과 보기".
- 영문/내부 용어 금지: "Set Up Model" · "Run Simulation"이 ko 화면에 그대로면 번역한다
  ("모델 설정" · "시뮬레이션 실행"). 영문 라벨이 코드에 하드코딩돼 i18n을 우회하는 건
  버그다 — 키로 빼라.

## 7. 내부·개발 용어를 사용자 언어로 (★ 자주 새는 곳)
패키지명·라이브러리·시스템 상태·CAD/엔지니어 jargon은 사용자에게 의미가 없다. 번역하거나
숨긴다. (에러의 #4 "사용자 입장 언어"와 같은 뿌리.)

| 내부 노출 (❌) | 사용자 언어 (✅) | 비고 |
|---|---|---|
| 워크벤치 엔진 다운로드 중 | 3D 작업 공간 준비 중 / 시뮬레이터 준비 중 | 패키지명·"engine" 노출 |
| OCCT 워크플로 연결 전… | (사용자 화면이면 문구 자체를 숨김) | 라이브러리명 노출 |
| 예외 처리된 신분증입니다 | 기간이 만료된 신분증이에요 | 시스템 상태어 |
| FaceID / Bonded / Solid(영문) | 면 / 접합 / 솔리드 | 미번역 jargon |
| Compiling / Booting | (단계 합쳐 "준비 중") | 부트스트랩 단계 노출 |

휴리스틱: **개발자만 아는 단어가 화면에 있으면 새고 있는 것.** grep으로 패키지명·라이브러리명·
영문 동사형 라벨을 훑어라.

## 8. 숫자·단위·표기 통일
- 단위는 사용자 언어로: "[m/s]"를 그대로 두기보다 맥락에 맞게(전문 도구면 단위 유지 가능).
- 한 개념은 한 단어로(용어집). 같은 화면에서 "유입 방향 / 유동 방향 / 방향"이 섞이면 하나로.
- 로케일 간 동기화: en "Velocity" ↔ ko "속도" — 한쪽만 "Flow Velocity"면 드리프트.

## 예외 표 — 기계적 치환 금지 (토스 가이드 명시)

| 규칙 | 예외 (이럴 땐 어겨도 됨) |
|---|---|
| 능동형 | **서비스 종료·만료**처럼 주체를 강조할 때 / **영향 경고**로 인과를 분명히 할 때 / 민감한 상황에서 사용자를 안심시킬 때 → 수동형 허용 |
| 높임말 절제 | 사용자의 **맥락·판단**을 묻거나 불확실한 상황, **설문·자발적 입력**엔 "~하시겠어요? · ~하시나요?" 허용 |
| 긍정형 | **정책 제약**을 분명히 해야 할 때 / **특정 기능 한계**를 명시할 때 / 민감 데이터 처리를 안심시킬 때 → 부정형 허용 |

> 규칙이 충돌하면 5 코어밸류(`principles.md`)로 판정한다. 의심스러우면 소리 내어 읽어라 —
> 입에 붙지 않으면 고친다.
