## 의도한 단순화

스코프·시간 상 의도적으로 **덮지 않은** 엣지 케이스. "알고 안 한 것"과 "모르고 안 한 것"을 구분해 드러낸다.

- 예: 서버 에러 재시도는 구현하지 않음. 이유: 요구사항에 없음 + 2시간 예산에 비해 설계가 무거워짐. 확장 지점은 `api.ts:X`에 `fetcher` 레이어를 끼울 수 있게 열어둠.

## 확장 지점 (Extensibility)

- 도메인이 늘어나면 어디를 건드려야 하는가 (`features/*` 추가)
- 상태가 복잡해지면 어디부터 분해하는가 (훅 단위 / 라우트 분리 / …)
- breaking 변경이 필요한 API는 무엇인가

## 러닝 포인트 (Learning)

이번 과제에서 **새로 배운 것** 또는 **기존 가정을 고친 것**. "토스는 이력서에서도 임팩트와 러닝 포인트를 구체적으로" 보라고 공식적으로 언급한다.

- (예) View Transitions API를 시도했다가 useEffect 기반 guard와 경합이 생겨 `afterTransition` 콜백 패턴으로 뒤집음. 교훈: 새 API 도입은 버그 없이 끝낼 시간이 있을 때만.
