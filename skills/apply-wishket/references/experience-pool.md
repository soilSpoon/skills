# 경험 풀 (Experience Pool)

master.yaml에서 추출한 사용 가능한 경험 목록. 지원서 작성 시 이 풀에서 2-3개를 선택한다.

> **이 파일은 master.yaml이 변경될 때마다 갱신해야 한다.**
> 최종 갱신: 2026-03-27

## 매칭 규칙

1. 프로젝트 요구사항의 키워드를 경험 풀과 대조하여 관련도 순으로 2-3개 선택
2. **동일 코드에서 2개 스토리 추출 금지** — HR에서 권한 이야기를 쓰면 같은 HR에서 성능 이야기를 쓰지 않음
3. 다건 처리 시 10건 전체에서 경험 배분이 편중되지 않도록 다양한 조합 사용
4. 각 경험은 반드시 "이슈→해결→결과" 3줄 구조
