# 지원서 작성 규칙

## 템플릿 (4단 구조)

```
안녕하세요. React/TypeScript 7년차 풀스택 개발자입니다.

[프로젝트 분석]
본 프로젝트의 핵심은 다음과 같이 파악됩니다:
1. (핵심 1)
2. (핵심 2)
3. (핵심 3)

예상되는 주요 이슈:
- (이슈 1): (왜 이슈인지 한 줄 설명)
- (이슈 2): (왜 이슈인지 한 줄 설명)

[유사 프로젝트 경험]
1) (경험 제목 — 익명화)
- (프로젝트 배경 한 줄)
- 실제로 겪은 이슈: (XX 문제) → (YY 방식으로 해결)
- 결과: (성과/효과)

2) (경험 제목 — 익명화)
- (프로젝트 배경 한 줄)
- 실제로 겪은 이슈: (XX 문제) → (YY 방식으로 해결)
- 결과: (성과/효과)

[진행 프로세스]
1단계. (단계명) — (확정 기간)
- (구체적 내용)
- (산출물/공유 항목)
- (중간 공유: 주 N회 진행 보고 / 중간 데모)

2단계. (단계명) — (확정 기간)
- (구체적 내용)
- (중간 데모/공유 포함)

3단계. (단계명) — (확정 기간)
- (구체적 내용)
- (최종 산출물)

[미팅 시 논의 희망 사항]
- (논의 사항 1)
- (논의 사항 2)
- (클라이언트가 미처 고려하지 못했을 수 있는 포인트 1개)

포트폴리오와 상세 경험은 프로필에서 확인 가능합니다.
감사합니다.
```

## DO

- 프로젝트별 맞춤 작성 — 복붙 금지, 프로젝트 고유 내용 필수
- 경험 중심 어필 — 기술 디테일보다 "무엇을 해서 어떤 결과를 냈는지"
- 개조식/명사형 종결 — "~했습니다" 반복보다 "~수행", "~해결"
- 830자 이상 (위시켓 데이터봇 권장)
- 포트폴리오 2-3개 첨부 (매칭 확률 60% 상승)
- 모든 경험에 "이슈→해결→결과" 패턴 필수 (나열 금지)
- "미팅 시 논의 희망 사항" 포함
- 기간은 확정형 ("2주") 사용
- **진행 프로세스에 중간 공유 포인트 명시** — "주 1회 진행 공유", "중간 데모 N회" 등 (성공 파트너 평가에서 "소통", "적극성"이 반복 언급)
- **클라이언트가 미처 생각하지 못한 포인트 1개 제안** — "논의 사항"에 포함 (성공 파트너 평가: "생각하지 못한 부분까지도 제안")

## DON'T

- 기술 스택 나열 섹션
- 컴포넌트/훅 이름 (PermissionGate, usePermission 등)
- 기술 버전 (AI SDK v5 등)
- 코드량 지표 (커밋 수, PR 수)
- 범위형 기간 ("1~2주", "2~3주")
- 기술 용어 직접 사용 — 비즈니스 언어로 전환:

| 금지 | 대체 |
|------|------|
| hospital_id, tenant_id | 병원별/테넌트별 데이터 격리 |
| RBAC | 역할별 접근 권한 체계 |
| Chart.js, D3 | 시각화 대시보드 |
| API 레이어 추상화 | 데이터 호출 구조 정리 |
| WebSocket | 실시간 데이터 연동 |
| 가상화 (virtualization) | 대량 데이터 화면 최적화 |
| React Query, Zustand, Jotai | (언급하지 않음) |

## 금액/기간 산정

단순 "예산 동일"이 아닌 5단계 분석:

**Step 1: 스코프 분석** — 기능 모듈 수, 업무 범위(개발/디자인/기획), MVP vs 풀 스코프

**Step 2: 공수 추정** — 모듈 × 복잡도 = 인일(man-day)

**Step 3: 일당 단가 계산** — 예산÷기간

**Step 4: 시장 + 벤치마크 비교**

시장 기준과 실제 성공 파트너 데이터를 모두 참조:

| 기준 | 일당 범위 | 근거 |
|------|----------|------|
| 시장 시니어 풀스택 | 25~40만/일 | 위시켓 일반 시세 |
| 성공 파트너 A (12건 계약, 5.0) | 25~30만/일 | 150만/6일 ~ 3000만/99일 |
| 성공 파트너 B (31건 계약, 5.0) | 9~23만/일 | 80만/9일 ~ 850만/70일 (볼륨형) |
| 성공 파트너 C (78건 계약, 4.93) | 다양 | 60만 ~ 1000만 (니치 도메인) |
| **신규 파트너 권장** | **25~33만/일** | 실적 0건 → 상한(40만)은 비현실적 |

핵심 인사이트:
- 성공 파트너도 초기에는 소규모(150만/6일)로 시작하여 실적 쌓은 후 대형(3000만/99일) 수주
- **위시켓 데이터봇: 미팅 성사 파트너의 금액이 평균보다 오히려 높았다** → 저가 경쟁은 역효과
- **예산보다 싸게 써서 계약까지 간 경우 14%뿐** → 클라이언트는 금액보다 유사 경험 중시

**Step 5: 최종 결정**

| 상황 | 전략 |
|------|------|
| feasibility 4.0+ + 일당 25~33만 | 예산 동일. 기간 10% 단축 가능 (자신감 어필) |
| feasibility 3.0~3.9 + 일당 25~33만 | 예산 동일, 기간 동일 |
| feasibility 3.0~3.9 + 일당 25만 미만 | 예산 동일 유지 + "미팅에서 스코프 우선순위 조율" 멘트 |
| feasibility 3.0~3.9 + 일당 40만 이상 | 기간 단축(-10~15%)으로 가치 어필 |
| feasibility 2.0~2.9 | MVP 범위 한정 명시 + 예산 동일 (저가 경쟁 금지) |
| **첫 수주 전략** | 500만~1500만 소규모에서 확실히 성공 → 5.0 리뷰 확보 → 단가 상향 |

## 성공 파트너 패턴 (지원서 내용에 반영)

위시켓 상위 파트너 3명의 클라이언트 평가에서 반복 등장하는 키워드:

| 패턴 | 빈도 | 지원서 반영 위치 |
|------|------|----------------|
| "적극적" / "적극성" | 모든 파트너 | 진행 프로세스: "주 N회 진행 공유" 명시 |
| "문제해결이 빠르다" | 파트너 A | 유사 경험: "이슈→해결" 패턴으로 어필 |
| "사전 미팅 준비성" / "꼼꼼" | 파트너 B | 프로젝트 분석: 이슈를 미리 정리해서 보여주기 |
| "생각하지 못한 부분까지 제안" | 파트너 A | 미팅 논의: 클라이언트가 놓칠 수 있는 포인트 1개 |
| "수정요청에도 끝까지" | 파트너 B | 진행 프로세스: QA/수정 단계 명시 |
| "클라이언트 입장 고려" | 파트너 B | 전체 톤: 기술 과시 아닌 비즈니스 문제 이해 |

이 패턴들은 템플릿에 이미 구조적으로 녹아있다:
- **[프로젝트 분석]** → "꼼꼼함", "준비성"
- **[유사 경험] "이슈→해결"** → "문제해결 속도"
- **[진행 프로세스] 중간 공유** → "적극성", "소통"
- **[미팅 논의] 추가 제안** → "생각하지 못한 부분 제안"
- **QA/수정 단계 포함** → "수정요청에도 끝까지"
