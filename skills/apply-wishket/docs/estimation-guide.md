# 공수 추정 가이드

프로젝트 설명을 깊이 분석하여 공수(인일)와 난이도를 산정하는 방법.
"대략 50인일" 같은 감이 아니라, 기능별로 분해하고 근거를 남긴다.

## Step 1: 기능 분해 (Feature Extraction)

프로젝트 설명에서 **구현해야 할 모든 기능**을 추출한다. 명시된 것뿐 아니라 암묵적으로 필요한 것도 포함.

추출 카테고리:

| 카테고리 | 예시 |
|---------|------|
| 인증/인가 | 회원가입, 로그인, 소셜 로그인, 권한 관리, 역할 분리 |
| 핵심 비즈니스 | 게시물 CRUD, 검색/필터, 매칭 로직, 예약, 결제 |
| 데이터 | 대시보드, 통계, 리포트, 차트, 엑셀 다운로드 |
| 외부 연동 | PG 결제, 알림톡/SMS, 외부 API, OAuth, 크롤링 |
| 관리자 | 관리자 페이지, 사용자 관리, 콘텐츠 관리, 설정 |
| 인프라 | 배포, CI/CD, 모니터링, 도메인/SSL |
| 모바일 | 앱 개발, 푸시 알림, 스토어 배포 |
| 디자인 | UI/UX 디자인, 반응형, 디자인 시스템 |
| 암묵적 필수 | 에러 처리, 로딩 상태, 폼 검증, 페이지네이션, 파일 업로드 |

**암묵적 기능을 놓치지 않는 것이 핵심.** 클라이언트가 "게시판"이라고 쓰면 실제로는 CRUD + 검색 + 페이지네이션 + 파일 첨부 + 권한 분리 + 관리자 페이지가 필요하다.

## Step 2: 기능별 복잡도 평가

각 기능에 복잡도를 부여한다:

| 복잡도 | 기준 | 인일 기준 |
|--------|------|----------|
| **S (단순)** | 정형화된 CRUD, 라이브러리만으로 구현 가능, 기존 경험 그대로 적용 | 1~2일 |
| **M (보통)** | 커스텀 로직 필요, 여러 모듈 연동, 엣지 케이스 있음 | 3~5일 |
| **L (복잡)** | 외부 API 연동, 실시간 처리, 복잡한 비즈니스 로직, 최적화 필요 | 5~10일 |
| **XL (매우 복잡)** | 미경험 기술, 연구 필요, 성능 크리티컬, 보안 민감 | 10~20일 |

### 복잡도 결정 체크리스트

기능 하나에 대해 아래 질문에 답한다:

```
1. 이 기능을 이전에 구현해본 적 있는가?
   → 있음: 복잡도 -1단계 / 없음: 그대로 또는 +1단계

2. 외부 시스템과 연동이 필요한가?
   → 필요: +1단계 (API 문서 파악, 에러 핸들링, 인증 등 추가 공수)

3. 실시간 처리가 필요한가? (WebSocket, SSE, 스트리밍)
   → 필요: +1단계

4. 데이터 양이 많은가? (수천~수만 건 이상)
   → 많음: +1단계 (최적화, 페이지네이션, 캐싱 필요)

5. 보안이 민감한가? (결제, 개인정보, 금융 데이터)
   → 민감: +1단계 (암호화, 감사 로그, 권한 검증)
```

## Step 3: 생산성 계수 (Tech Match)

우리의 기술 스택 매칭도에 따라 생산성이 달라진다:

| 매칭도 | 생산성 계수 | 의미 |
|--------|-----------|------|
| 동일 스택 실무 경험 | ×1.0 | 기본 인일 그대로 |
| 유사 스택 경험 | ×1.3 | 학습+적응 시간 30% 추가 |
| 새로운 스택 | ×1.5~2.0 | 학습 시간 상당, 삽질 가능성 |

우리의 핵심 스택 (×1.0 적용):
- React, TypeScript, Next.js, Tailwind
- Laravel (PHP), NestJS (Node.js)
- MySQL, PostgreSQL
- React Native, Electron

생산성 페널티 (×1.3~2.0 적용):
- Vue, Angular, Svelte → ×1.3
- Django, Flask, Spring → ×1.5
- Flutter, Swift, Kotlin native → ×1.5
- WebRTC, 음성/영상 처리 → ×2.0
- 블록체인, 스마트 컨트랙트 → ×2.0

## Step 4: 총 공수 산정

```
총 공수(인일) = Σ (기능별 인일 × 생산성 계수) + QA/수정 버퍼

QA/수정 버퍼 = 기능 합계 × 0.2 (전체의 20%)
```

## Step 5: 검증 — 상식 체크

산정된 공수가 합리적인지 크로스 체크:

| 프로젝트 규모 | 기대 공수 범위 | 예시 |
|-------------|-------------|------|
| 소규모 (기능 3~5개, S~M) | 10~25인일 | 기업 홈페이지, 간단한 관리자 |
| 중규모 (기능 6~10개, M~L) | 25~60인일 | SaaS MVP, 매칭 플랫폼 MVP |
| 대규모 (기능 10개+, L~XL) | 60~120인일 | 풀 플랫폼, 멀티 앱 |
| 초대규모 (기능 15개+) | 120인일+ | 1인 수행 재고 필요 |

추가 상식 체크:
- 1인 개발자가 1달에 산출할 수 있는 공수: ~22인일 (주 5일 × 4.3주)
- 90일(3달) 프로젝트 가용일: ~64일
- 공수 > 가용일 × 1.0이면 야근/주말 불가피 → 강도 분석으로 연결
