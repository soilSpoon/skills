## 공통 패턴 분석

### 프로필 구조
1. **자기소개**: 2-3개 키워드 태그 + 1문단 요약
2. **경력**: 회사명 → 직무 → 기간 → 기술스택 태그 → 업무 설명
3. **직무 관련 활동**: 프로젝트/오픈소스/교육 등
4. **학력** / **외국어** / **관심 스킬**
5. **이력서 하이라이트**: 그룹바이가 뽑은 돋보이는 표현 + 작성 가이드

### 그룹바이가 뽑은 돋보이는 패턴

| 카테고리 | 돋보이는 표현 예시 | 작성 가이드 |
|---------|----------------|-----------|
| 강점 어필 | "대규모 트래픽 처리와 복잡한 비즈니스 로직 구현에 강점" | 가장 잘하는 업무를 초반에 명확히 제시 |
| 문제 해결 | "기존 문제: X → 해결: Y → 성과: Z" 구조 | before/after 서사로 문제 해결 과정을 보여줌 |
| 주도성 | "비효율성을 절실히 느꼈고, 이를 해결하기 위해 프로그래밍을 시작" | 커리어 방향을 직접 느낀 문제와 연결 |
| 기술 활용 | "TypeScript와 NestJS의 강력한 타입 시스템을 활용하여" | 기술스택 사용 경험을 구체적으로 제시 |
| 개발 철학 | "나중에 변경이 생겨도 쉽게 대응할 수 있는 구조를 고민" | 시키는 대로가 아닌 가치관이 있음을 보여줌 |
| 성장 의지 | "뛰어난 동료들과 함께 빠르게 성장하기" | 빠르게 성장할 수 있는 사람임을 표현 |
| 업무 범위 | "머신러닝에 국한되지 않고 필요한 역할을 폭넓게 수행" | 하나만 하는 게 아닌 다양한 범위를 보여줌 |

### 합격자 공통 특징
- **성과를 수치로 표현**: "검색 응답 시간 800ms → 150ms (81% 향상)", "배포 시간 95% 감소"
- **문제 → 해결 → 성과 서사**: 단순 나열이 아닌 스토리텔링
- **구체적 기술 선택 이유**: "왜 이 기술을 선택했는지"가 드러남
- **짧은 자기소개**: 장황한 설명 대신 핵심 강점 1-2줄로 압축
