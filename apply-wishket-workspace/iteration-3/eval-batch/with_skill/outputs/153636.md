## 지원서 본문

[프로젝트 분석]
본 프로젝트의 핵심은 다음과 같이 파악됩니다:
1. 기존 코드 이어받기 — 이전 개발자의 설계 의도와 미완성 부분을 정확히 파악한 후 개발을 이어가야 하며, 무리한 리팩토링보다 안정적 론칭이 우선
2. 채팅 서비스 안정성 확보 — AI 캐릭터와의 대화 흐름이 끊기거나 응답이 누락되면 사용자 이탈에 직결
3. 결제 시스템 연동 — 토스페이먼츠 PG 연동을 통해 유료 서비스 전환의 기반 마련

예상되는 주요 이슈:
- 기존 코드 파악 비용: 전임 개발자의 문서화 수준에 따라 코드 이해에 시간이 달라짐 — 초기 2~3일을 코드 분석에 집중 배정하여 리스크 최소화 필요
- 채팅 상태 관리 복잡성: AI 응답 스트리밍, 대화 히스토리, 캐릭터 컨텍스트 전환 등이 얽히면 상태 꼬임 발생 가능 — 상태 흐름을 먼저 정리한 후 기능 추가

[유사 프로젝트 경험]
1) SaaS 플랫폼 — AI 챗봇으로 복잡한 워크플로우 진입 장벽 제거
- 124개국 사용자가 이용하는 시뮬레이션 플랫폼에서, 5단계 파이프라인에 전문 지식이 필요하여 비전문가 독립 사용이 불가한 상태
- 실제로 겪은 이슈: 각 단계에 전문 용어와 설정 지식이 필요 → AI 챗봇을 구축하여 자연어로 파이프라인을 직접 제어할 수 있는 인터페이스 구현. 스트리밍 응답 렌더링, 대화 히스토리, 도구 실행 상태 관리까지 구축
- 결과: 비전문가도 자연어로 시뮬레이션 조작 가능 — 지식 장벽 대폭 감소. 이 경험이 본 프로젝트의 AI 캐릭터 채팅 안정화, 스트리밍 응답 처리, 대화 상태 관리에 직접 적용 가능

2) HR SaaS — jQuery 레거시에서 React 기반 상용 아키텍처로 전환
- 근태/급여 관리 SaaS에서 jQuery + PHP 5 기반 코드를 이어받아 React + Laravel 체계로 전환해야 하는 상황. 비즈니스 로직과 뷰가 한 파일에 혼재, 동일 UI가 페이지마다 제각각 구현된 상태
- 실제로 겪은 이슈: 기존 코드의 상태 관리 패턴이 혼재하여 수정 시 10곳 이상 일일이 수정 필요 → 컴포넌트 기반 UI 체계 구축 후 53개 공통 컴포넌트를 표준화, 서버·전역·로컬 3계층 상태관리로 전환
- 결과: 신규 페이지 개발 시간 50% 단축, 이 아키텍처 위에 6개 모듈 안정적 신규 개발 완료. 기존 코드를 이어받아 체계적으로 전환한 경험이 본 프로젝트의 코드 인수인계 + 리팩토링에 직접 활용 가능
