## 지원서 본문

안녕하세요. React/TypeScript 7년차 풀스택 개발자입니다.

[프로젝트 분석]
본 프로젝트의 핵심은 다음과 같이 파악됩니다:
1. 출제자-학생-관리자 3계층 권한 분리와 콘텐츠 승인/반려 워크플로우
2. PG 결제 → 워터마크 PDF 다운로드로 이어지는 디지털 콘텐츠 판매 파이프라인
3. 향후 과목 확장을 고려한 유연한 카테고리·메타데이터 DB 구조

예상되는 주요 이슈:
- 콘텐츠 승인 워크플로우 복잡성: 출제자 등록 → 관리자 검토 → 승인/반려 → 판매 상태 전환까지 상태 관리가 복잡해지기 쉬움. 상태 전이 규칙을 명확히 설계하지 않으면 "승인했는데 안 보인다" 류의 운영 이슈 발생
- PDF 워터마크 처리 성능: 결제 직후 실시간으로 워터마크를 삽입해 다운로드를 제공해야 하는데, PDF 처리가 동기적이면 결제 완료 후 대기 시간이 길어짐. 비동기 처리 + 상태 알림 설계 필요
- 정산 정확성: 출제자 수익 정산은 결제 취소·환불·부분 환불 케이스까지 고려해야 하며, 한 번이라도 정산 오류가 발생하면 신뢰 손상

[유사 프로젝트 경험]
1) HR 관리 SaaS — 3계층 권한 체계 + 관리자 시스템 4년 운영
- B2B 근태·급여 관리 SaaS를 4년간 설계·운영한 프로젝트
- 실제로 겪은 이슈: 어드민→사업주→관리자→근로자 다단계 권한이 코드 곳곳에 산재 → 역할별 접근 권한을 중앙에서 선언적으로 관리하는 구조로 재설계
- 결과: 이 아키텍처 위에 노무법인 시스템 6개 모듈을 안정적으로 신규 개발 완료, 신규 권한 추가 시 설정만 변경

2) 제철소 실시간 품질 분석 시스템 — 규칙 엔진 기반 판정 체계
- 제철소 MES에서 철판 품질 등급을 자동 판정하는 시스템을 개발한 프로젝트
- 실제로 겪은 이슈: 공정마다 판정 기준이 달라 조건 분기가 점점 복잡 → 판정 로직을 규칙 엔진으로 추상화하여 설정 파일 기반으로 전환
- 결과: 코드 수정 없이 설정 파일만으로 신규 공정 판정 기준 대응 가능. 본 프로젝트의 콘텐츠 검수 기준(과목별, 학년별 메타데이터 검증)에도 동일한 접근 적용 가능
