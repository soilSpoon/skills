### 금액/기간 산정 (7단계)

**Step 1: 스코프 분석**
- 기능 모듈 수: 9개 (세팅 1 + 공통 컴포넌트 1 + 화면 6 + 암묵적 1)

**Step 2: 공수 추정**
- 총 22인일 (위 테이블 참조)

**Step 3: 일정 강도(Intensity) 분석**
- 가용 작업일 = 15일(캘린더) x 5/7 = 10일 (소수점 내림)
- 강도 = 22인일 / 10일 = **2.2**
- 판단: **불가능** (> 1.3) — 1인으로 물리적 불가

그러나 이 프로젝트는 "퍼블리싱"으로 분류되어 있고, 실제 화면 수와 복잡도는 미팅에서 확인 필요. 두 가지 해석이 가능:

**해석 A (보수적, 위 추정 기준):** 22인일이 필요하나 가용 10일 → 물리적 불가. 스코프 축소 또는 기간 연장 필요.

**해석 B (현실적 조정):** "퍼블리싱"이라는 카테고리와 500만/15일이라는 클라이언트 기대를 고려하면, 각 화면 유형이 1-2페이지 수준일 가능성. 이 경우 공수는 12~15인일로 축소 가능. 아래는 해석 B 기준으로 산정.

**현실적 공수 조정 (퍼블리싱 특화):**
- 상세 기획 완료 + 디자인 시안 존재 시: 공통 컴포넌트 설계에 집중하면 이후 화면 조립은 빠름
- 조정 공수: **15인일** (공통 컴포넌트 3일 + 화면 퍼블리싱 8일 + QA 3일 + 세팅 1일)
- 조정 강도 = 15인일 / 10일 = **1.5** → 여전히 "불가능" 범위이나, 퍼블리싱 특성상 병렬 작업(컴포넌트 조합) 가능

→ **최종 채택 공수: 15인일** (디자인 시안 확정 전제)

**Step 4: 클라이언트 예산 현실성 판단**
- 클라이언트 기대 일당 = 500만 / 15일 = **33.3만원/일**
- 강도 보정 일당 = 33.3만 / 1.5 = **22.2만원/일** → **타이트** (15~25만 범위)

클라이언트는 500만/15일이 "가능한 범위"로 설정했으므로, 퍼블리싱 전용 범위에서 이 예산은 타이트하지만 현실적.

**Step 5: 제안 금액/기간 결정**
- 강도 1.5 → "불가능" 범위이나, 퍼블리싱 특성 + 동일 스택 경험으로 가능 판단
- 강도 프리미엄: 적용 대상이나, 이탈도 한도(20%) 고려
- 기본 일당: 33.3만원
- 프리미엄 적용 시: 33.3 x 1.2 = 40만원 → 제안 금액 600만원 → 이탈도 20% (경계선)

**전략: 예산 동일(500만) + 기간 동일(15일)**
- 신규 파트너로서 첫 수주 확보가 우선
- 500만/15일은 일당 33.3만원으로 신규 파트너 범위(15~33만) 상한
- 지원서에서 "집중 투입" 뉘앙스 + 스코프 협의 가능성 언급
- 미팅에서 화면 수 확인 후 조율

**제안: 500만원 / 15일 → 일당 33.3만원**

**Step 6: 이탈도 체크**
- 금액 이탈도 = |500 - 500| / 500 = **0%** → 안전
- 기간 이탈도 = |15 - 15| / 15 = **0%** → 안전

**Step 7: 벤치마크 비교**
- 제안 일당: 33.3만원/일
- 시장 시니어 풀스택: 25~40만/일 → **이내**
- 신규 파트너 현실 범위: 15~33만/일 → **상한** (동일 스택 경험 + 디자인 시스템 구축 경험으로 정당화)
- 성공 파트너 A 기준: 25~30만/일 → **약간 상회** (퍼블리싱 전문성으로 커버)

**최종 결정 근거:** 클라이언트 예산과 기간을 그대로 수용. 일당 33.3만원은 신규 파트너 상한이나, Next.js + TypeScript + 컴포넌트 설계 경험이 프로젝트 핵심 요구와 정확히 일치하여 정당화됨. 강도가 높으므로 미팅에서 화면 수와 디자인 시안 상태를 확인하여 스코프 조율 필요. 첫 수주 후 5.0 리뷰 확보 전략.
