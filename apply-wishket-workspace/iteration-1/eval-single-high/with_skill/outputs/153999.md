## 지원서 본문

안녕하세요. React/TypeScript 7년차 풀스택 개발자입니다.

[프로젝트 분석]
본 프로젝트의 핵심은 다음과 같이 파악됩니다:
1. Figma 디자인 시안 + 디자인 토큰 가이드를 Next.js App Router 기반 컴포넌트로 정확하게 구현
2. 조회(검색+테이블), 마스터-디테일, 설정(탭), 통계 등 어드민 화면 유형별 재사용 가능한 컴포넌트 체계 설계
3. 사용자 계정 관리·조직 연동 등 B2B 특유의 데이터 구조를 반영한 폼/테이블 설계

예상되는 주요 이슈:
- 화면 유형 간 공통 패턴 추출 시점: 조회·마스터-디테일·설정 화면이 테이블·폼·모달을 공유하므로, 초기에 공통 컴포넌트를 잘 설계하지 않으면 후반부 화면에서 중복 구현이 누적
- 디자인 토큰과 Tailwind 매핑: Figma 토큰 체계를 Tailwind config에 정확히 옮기지 않으면 화면마다 미세한 스타일 불일치 발생

[유사 프로젝트 경험]
1) B2B SaaS 어드민 — 디자인 시스템 53개 공통 컴포넌트 구축
- 근태·급여 관리 SaaS의 어드민 프론트엔드를 4년간 설계·운영한 경험
- 실제로 겪은 이슈: 동일한 테이블·폼이 페이지마다 제각각 구현되어 수정 시 10곳 이상을 일일이 수정해야 하는 비효율 → 프로젝트 전용 53개 공통 컴포넌트를 추출·표준화하여 디자인 시스템 구축
- 결과: 공통 컴포넌트 조합만으로 신규 페이지 구성 — 개발 시간 50% 단축, 이 체계 위에 6개 모듈 안정적 신규 개발 완료

2) 시뮬레이션 플랫폼 — Next.js App Router + 디자인 시스템 표준화
- Next.js App Router 기반 B2B SaaS 플랫폼의 프론트엔드 설계·개발
- 실제로 겪은 이슈: 프로토타입 수준의 UI에 일관성이 없고 컴포넌트 재사용 구조 부재 → 통합 컬러 팔레트 시스템 + 컴포넌트 라이브러리 재구성으로 디자인 시스템 표준화, 컴포넌트 단위 개발·테스트 환경 구축
- 결과: UI 일관성 확보 및 신규 화면 개발 속도 향상, 안정적인 v1.0 릴리즈 완료
