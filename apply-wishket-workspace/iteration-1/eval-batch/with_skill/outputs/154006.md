## 지원서 본문

[프로젝트 분석]
본 프로젝트의 핵심은 다음과 같이 파악됩니다:
1. 출제자-학생 양면 플랫폼 구조 — 콘텐츠 등록/검수/판매/다운로드 전 과정을 하나의 서비스에서 처리해야 하며, 역할별 권한 분리가 중요
2. 과목 확장을 고려한 DB 설계 — 현재 국어/사회/과학이지만 향후 확장 가능한 유연한 스키마 필요
3. 결제 연동 및 디지털 콘텐츠 보호 — PG사 결제, 워터마크 처리, 다운로드 관리 등 이커머스 핵심 기능

예상되는 주요 이슈:
- 역할별 권한 경계: 출제자/검수자/관리자/학생 각각 접근 가능한 데이터와 기능이 달라, 권한 체계를 초기에 잘못 설계하면 이후 모든 기능에 영향
- 콘텐츠 검수 워크플로우: 등록→검수→승인→판매 상태 전이가 명확해야 하며, 반려/수정 흐름까지 고려한 설계 필요

[유사 프로젝트 경험]
1) B2B SaaS 플랫폼 4년 운영 — 역할별 권한 체계 및 디자인 시스템 구축
- 근태/급여 관리 SaaS를 4년간 설계·운영한 프로젝트
- 실제로 겪은 이슈: 5단계 권한(어드민→노무법인→사업주→관리자→근로자) 분기 로직이 곳곳에 산재하여 신규 역할 추가 시 전체 페이지 검토 필요 → 선언적 권한 기반 렌더링으로 중앙 집중화, API 레벨 이중 검증 적용
- 결과: 권한 로직 중앙 집중화로 노무법인 시스템 6개 모듈을 안정적으로 신규 개발 완료, 신규 권한 레벨 추가 시 설정 변경만으로 대응

2) 시뮬레이션 플랫폼 — 프로토타입에서 상용 서비스로 전환
- 드론/시뮬레이션 SaaS를 상용 수준으로 재설계한 프로젝트
- 실제로 겪은 이슈: 프로토타입 수준 코드베이스를 상용 제품으로 전환해야 했으나 UI 일관성 부재와 인증 미비 → 디자인 시스템 표준화 + 인증 체계 구축 + 모듈 간 의존성 분리로 v1.0 릴리즈
- 결과: 디자인 시스템 표준화로 UI 일관성 확보 및 신규 화면 개발 속도 향상, v1.0 릴리즈 후 다국어 서비스 운영 중
