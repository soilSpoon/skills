[프로젝트 분석]
본 프로젝트의 핵심은 다음과 같이 파악됩니다:
1. 출제자-학생-관리자 3자 역할 기반 콘텐츠 마켓플레이스 구조 설계
2. PG 결제 연동 + PDF 워터마크 처리 등 콘텐츠 보호/유통 파이프라인
3. 콘텐츠 등록→검수→승인→판매→정산까지 전 과정의 체계적 워크플로우

예상되는 주요 이슈:
- 역할별 접근 권한 경계: 출제자/학생/관리자가 각각 다른 데이터에 접근해야 하며, 콘텐츠 승인 흐름에 따라 권한이 동적으로 변경되는 구조 — 초기 설계가 부실하면 이후 역할 추가 시 전체 페이지 수정 필요
- PDF 콘텐츠 보호: 워터마크 삽입, 다운로드 제한, 미리보기 범위 제한 등 디지털 콘텐츠 유출 방지 장치가 결제 전후로 다르게 작동해야 함
- 정산 데이터 정합성: 결제-취소-환불-정산 흐름에서 금액 불일치가 발생하면 출제자 신뢰 하락 — 트랜잭션 일관성 확보 필수

[유사 프로젝트 경험]
1) B2B SaaS 근태/급여 관리 플랫폼 — 역할별 권한 체계
- 어드민→법인→사업주→관리자→근로자 5단계 권한이 존재하는 B2B2C SaaS를 4년간 설계·운영
- 실제로 겪은 이슈: 권한 분기 로직이 페이지마다 산재하여 새 역할 추가 시 전체 코드 검토 필요 → 선언적 권한 기반 렌더링으로 중앙 집중화하여 해결
- 결과: 이 구조 위에 노무법인 시스템 6개 모듈을 안정적으로 신규 개발 완료, 신규 역할 추가 시 설정만 변경

2) 시뮬레이션 플랫폼 — 결제/유통 + 데이터 기반 의사결정
- 124개국 사용자가 이용하는 SaaS 플랫폼에서 사용자 행동 분석 파이프라인 구축
- 실제로 겪은 이슈: 제품 방향성 결정이 감에 의존하는 상태에서 사용자 데이터가 전혀 없음 → 클라이언트·서버 양쪽 이벤트 추적 파이프라인을 처음부터 구축
- 결과: 사용자 이탈 단계를 정량적으로 파악하여 데이터 기반 제품 방향성 결정 체계 확립
