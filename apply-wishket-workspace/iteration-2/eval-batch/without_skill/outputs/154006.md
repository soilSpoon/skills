## 프로젝트 이해

문제은행 웹 서비스는 학습 자료(국어, 사회, 과학) 거래 플랫폼으로, 학생(구매자)·출제자(판매자)·관리자 세 역할을 중심으로 콘텐츠 등록-검수-결제-다운로드 흐름을 구현해야 합니다. 핵심은 PG 결제 연동, PDF 워터마크 처리, 그리고 향후 과목·기능 확장을 고려한 DB 설계입니다.

---

## 적합성

### 1. 풀스택 웹 플랫폼 구축 경험

**출근노트 (B2B SaaS, 4년 설계·운영)**
- jQuery/PHP 5 레거시에서 React + Laravel로 전면 전환, 프로젝트 초기 세팅부터 운영까지 전 과정을 직접 수행
- Chakra UI 기반 디자인 시스템 53개 공통 컴포넌트 구축 — 신규 페이지 개발 시간 50% 단축
- B2B2C 5단계 권한 체계(어드민→노무법인→사업주→중간관리자→근로자) 설계 및 PermissionGate 패턴으로 중앙 집중화

본 프로젝트의 학생/출제자/관리자 3단계 권한 체계는 이보다 단순한 구조로, 동일한 패턴을 적용하여 빠르게 구현 가능합니다.

### 2. PG 연동 이커머스 경험

출근노트에서 결제 모듈 연동 경험이 있으며, 결제 흐름(주문 생성 → PG 승인 → 웹훅 검증 → 상태 갱신) 전체를 이해하고 있습니다. Toss Payments, 아임포트(포트원) 등 국내 PG사 연동에 익숙합니다.

### 3. PDF 처리 및 콘텐츠 플랫폼

- 서버 사이드 PDF 워터마크 삽입은 Node.js의 pdf-lib 또는 Laravel 환경의 FPDI/TCPDF로 구현 가능하며, 구매자 정보(이름, 주문번호)를 워터마크로 삽입하여 무단 배포를 방지하는 구조를 제안합니다
- 출제자의 콘텐츠 등록 → 관리자 승인 → 공개 흐름은 출근노트의 권한 기반 상태 관리 패턴을 그대로 활용할 수 있습니다

### 4. 확장 가능한 아키텍처 설계

- React Query/Zustand/Jotai 3계층 상태관리 아키텍처 설계 경험 — 서버·전역·로컬 상태 분리로 확장 시에도 구조적 안정성 유지
- 수천 명 규모 데이터 처리 최적화 경험 (react-window 가상화, Redis 캐싱, N+1 해소)
- Self-hosted → ARC 전환, CI 3단계 병렬화, PHPStan 정적 분석 자동화 등 DevOps 경험

## 기술 스택 제안

| 영역 | 기술 | 선택 이유 |
|------|------|-----------|
| Frontend | Next.js + TypeScript | SSR/SSG로 SEO 확보, 반응형 웹 |
| UI | Tailwind CSS + shadcn/ui | 빠른 UI 개발, 일관된 디자인 시스템 |
| Backend | Nest.js (또는 Next.js API Routes) | 구조적인 모듈 설계, TypeScript 일원화 |
| DB | PostgreSQL | JSON 타입 지원, 확장성 |
| 결제 | Toss Payments (또는 포트원) | 국내 PG 연동 안정성 |
| PDF | pdf-lib (Node.js) | 서버 사이드 워터마크 삽입 |
| 파일 저장 | AWS S3 (또는 Cloudflare R2) | 비용 효율적 파일 관리 |
| 배포 | Vercel + Docker | 프론트 CDN + 백엔드 컨테이너 |

클라이언트 측에서 선호하시는 기술 스택이 있다면 유연하게 조율 가능합니다.
