## 지원서 본문

안녕하세요. React/TypeScript 7년차 풀스택 개발자입니다.

[프로젝트 분석]
본 프로젝트의 핵심은 다음과 같이 파악됩니다:
1. B2B SaaS 관리자 어드민의 고품질 UI 컴포넌트 체계 구축 — 버튼, 입력 폼, 테이블, 모달 등 재사용 가능한 공통 컴포넌트 설계
2. 조회·마스터-디테일·설정·통계 등 다양한 화면 유형에 대한 일관된 레이아웃 체계 확립
3. 내부 관리자가 사용자 계정, 조직, 이용 이력을 직관적으로 관리할 수 있는 UX 설계

예상되는 주요 이슈:
- 컴포넌트 범위 확정: 화면 수가 많을 경우 15일 내에 모든 화면을 높은 품질로 완성하려면 공통 컴포넌트 우선 정의 후 화면 조립 순서가 중요
- 테이블·통계 화면의 데이터 구조 협의: 실제 API 연동 전이라도 데이터 형태를 미리 정의해야 컴포넌트 구조가 흔들리지 않음
- 조직 연동 관리의 계층 구조: 조직 트리 깊이와 권한 범위에 따라 UI 복잡도가 크게 달라짐

[유사 프로젝트 경험]
1) B2B SaaS 어드민 — 53개 공통 컴포넌트 디자인 시스템 구축
- 근태·급여 관리 SaaS의 프론트엔드를 4년간 설계·운영
- 실제로 겪은 이슈: 동일한 테이블·폼이 페이지마다 제각각으로 구현되어 수정 시 10곳 이상을 일일이 고쳐야 하는 비효율 → 53개 공통 컴포넌트를 추출·표준화하여 디자인 시스템 구축
- 결과: 신규 페이지를 공통 컴포넌트 조합만으로 구성, 개발 시간 50% 단축. 이 체계 위에서 6개 신규 모듈을 안정적으로 추가 완료

2) Next.js App Router 기반 SaaS 플랫폼 개발
- 클라우드 기반 시뮬레이션 SaaS 프론트엔드를 Next.js App Router로 구축
- 실제로 겪은 이슈: 프로토타입 수준 코드베이스를 상용 제품으로 전환해야 하는 상황 → 통합 디자인 시스템 표준화, 컴포넌트 단위 개발·테스트 환경 구축
- 결과: UI 일관성 확보 및 신규 화면 개발 속도 향상, 안정적으로 v1.0 릴리즈 완료

- 사용자 계정 관리, 조직 연동 관리 화면 구현

- 최종 산출물: 완성된 퍼블리싱 코드 + 컴포넌트 구조 문서
